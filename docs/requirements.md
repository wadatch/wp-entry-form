## 1. 背景・目的

ボランティア応募をはじめとする各種「申し込み・応募・問い合わせ」を受け付けるための、**汎用入力フォーム**を提供する WordPress プラグイン。

既存の Contact Form 系プラグインに対する差別化ポイント:

1. **柔軟なフォーム定義** — 管理画面のフォームビルダーで、項目（フィールド）を自由に追加・編集・並べ替えできる。
2. **送信データの DB 保存** — メール送信して終わりではなく、応募内容を WordPress の DB（専用テーブル）に蓄積し、管理画面で一覧・検索・CSV エクスポートできる。
3. **応募フォームに必要な機能を標準装備** — 確認画面、ファイル添付、同意チェック、自動返信メール、スパム対策。

## 2. スコープ

### 2.1 v0.1.0 で実装する範囲（イン・スコープ）

- 管理画面のフォームビルダー（複数フォーム対応）
- ショートコード `[entry_form id="N"]` と Gutenberg ブロックによる埋め込み
- 専用テーブルへの送信データ保存
- 確認画面（フォームごとに有無を選択）
- ファイル添付フィールド
- 管理者通知メール + 応募者への自動返信メール
- スパム対策（ハニーポット + 送信レート制限）
- プライバシーポリシー同意チェックフィールド
- 送信一覧の閲覧・検索・ステータス管理・CSV エクスポート・手動削除

### 2.2 将来検討（アウト・オブ・スコープ／拡張余地）

- reCAPTCHA / トークン有効期限などの追加スパム対策
- 個人情報の保有期間に基づく**自動削除**（v0.1 は手動削除のみ）
- 条件分岐（特定の選択でフィールドを出し分け）
- 複数ページ（ステップ）フォーム
- 決済・外部 CRM 連携
- 入力途中の自動保存

## 3. 想定ユーザー

| ロール | 説明 | 主な操作 |
| --- | --- | --- |
| サイト管理者 | フォームを作成・管理する WordPress 管理者 | フォームの作成・編集、送信データの確認・エクスポート、設定 |
| 応募者（エンドユーザー） | 公開ページのフォームから応募する訪問者 | フォーム入力、確認、送信 |

## 4. 機能要件

### FR-1 フォームビルダー（管理画面）
- FR-1.1 フォームを複数作成・編集・複製・削除できる。
- FR-1.2 フォームに項目（フィールド）を追加・編集・削除・並べ替え（ドラッグ）できる。
- FR-1.3 各フィールドにラベル、必須/任意、プレースホルダ、補足説明、初期値を設定できる。
- FR-1.4 選択系フィールド（セレクト/ラジオ/チェックボックス）に選択肢を設定できる。
- FR-1.5 フォーム単位で確認画面の有無、成功メッセージ、ボタン文言、リダイレクト先を設定できる。

### FR-2 フィールド型
v0.1 で対応する型（詳細は設計書 §5）:
- 入力系: テキスト、テキストエリア、メール、電話番号、URL、数値、日付
- 選択系: セレクト（ドロップダウン）、ラジオ、チェックボックス（複数）
- 特殊: 同意チェック（必須化可能な単一チェック）、ファイル添付
- 表示系: 見出し、説明文（入力を伴わない装飾要素）

### FR-3 フォーム表示
- FR-3.1 ショートコード `[entry_form id="N"]` でページ・投稿・ウィジェットに埋め込める。
- FR-3.2 Gutenberg ブロックでフォームを選択して配置できる（ブロックエディタ上でフォーム一覧から選択）。
- FR-3.3 未公開/存在しないフォーム ID の場合は管理者にのみ警告を表示し、一般訪問者には何も表示しない。

### FR-4 入力〜送信フロー
- FR-4.1 確認画面ありの場合: 入力 → 確認 → 完了 の3ステップ。確認画面で「戻る」を押すと入力内容を保持して入力画面へ戻れる。
- FR-4.2 確認画面なしの場合: 入力 → 完了 の2ステップ。
- FR-4.3 バリデーションエラー時は、入力済みの値を保持したまま該当フィールドにエラーを表示する。
- FR-4.4 JavaScript 無効環境でも通常のフォーム POST で送信が完結する（プログレッシブ・エンハンスメント）。

### FR-5 バリデーション
- FR-5.1 必須チェック、メール形式、数値、URL、最大文字数、選択肢の妥当性を検証する。
- FR-5.2 同意チェックフィールドは未チェックの場合に送信を拒否する。
- FR-5.3 ファイルは拡張子・MIME タイプ・最大サイズを検証する。

### FR-6 データ保存
- FR-6.1 送信内容を専用テーブルに保存する（フィールドキー → 値の JSON）。
- FR-6.2 送信日時、IP アドレス、ユーザーエージェント、リファラ、ログインユーザー ID（あれば）を記録する。
- FR-6.3 添付ファイルは公開ディレクトリから直接アクセスできない場所に保存し、メタ情報を DB に記録する。

### FR-7 メール通知
- FR-7.1 管理者通知メール: 宛先（複数可）、件名、本文を設定でき、本文中で `{フィールドキー}` の差し込みができる。
- FR-7.2 自動返信メール: 応募者のメールアドレスを格納するフィールドを指定して送信。件名・本文・差出人名/アドレスを設定できる。
- FR-7.3 いずれもフォーム単位で有効/無効を切り替えられる。
- FR-7.4 メール送信に失敗してもデータ保存は成立させ、管理者が後から確認できるようにする。

### FR-8 送信データ管理（管理画面）
- FR-8.1 フォーム別に送信一覧を閲覧でき、ステータス（未読/既読/スパム/ゴミ箱）で絞り込める。
- FR-8.2 キーワード検索ができる。
- FR-8.3 個別送信の詳細（全フィールド値・添付ファイル・メタ情報）を表示できる。
- FR-8.4 CSV エクスポートができる。
- FR-8.5 送信データを手動で削除できる（添付ファイルも併せて削除）。

### FR-9 スパム対策
- FR-9.1 ハニーポット: 人間には見えないダミー項目に入力があった送信を拒否する。
- FR-9.2 送信レート制限: 同一 IP からの短時間の連続送信を制限する（回数・時間窓を設定可能）。

### FR-10 プライバシー
- FR-10.1 プライバシーポリシーへの同意チェックフィールドをフォームに含められる。
- FR-10.2 送信データの手動削除に対応する（自動削除は将来対応）。

## 5. 非機能要件

| ID | カテゴリ | 要件 |
| --- | --- | --- |
| NFR-1 | 動作環境 | WordPress 6.0 以上、PHP 7.4 以上 |
| NFR-2 | セキュリティ | nonce 検証、権限チェック、入力サニタイズ／出力エスケープ、`$wpdb->prepare` 利用、添付ファイルの直リンク防止 |
| NFR-3 | 国際化 | 全文言を翻訳対応（text domain: `wp-entry-form`）。初期 UI は日本語 |
| NFR-4 | 権限 | 管理操作は `manage_options` を既定とし、将来カスタム権限へ拡張可能な設計 |
| NFR-5 | 拡張性 | フィルター/アクションフックで挙動を拡張可能にする |
| NFR-6 | データ保全 | プラグイン無効化ではデータを残し、アンインストール時に削除するか設定で選べる |
| NFR-7 | アクセシビリティ | label と入力要素の関連付け、必須表示、エラーメッセージの明示 |

## 6. 制約・前提

- 単一サイト構成を主対象とする（マルチサイトは将来検討）。
- メール送信は WordPress の `wp_mail()` に依存する（SMTP プラグイン併用可）。
- ファイルアップロードのサイズ上限はサーバの PHP 設定（`upload_max_filesize` 等）に従う。

## 7. 受け入れ基準（抜粋）

- 管理画面でフィールドを5種以上組み合わせたフォームを作成し、公開ページに表示できる。
- 確認画面ありフォームで、入力→確認→送信が完了し、データが DB に保存される。
- 管理者通知と自動返信メールの両方が届く。
- ハニーポットに入力した送信が拒否される。
- 送信一覧から CSV をエクスポートでき、添付付き送信を削除するとファイルも消える。

## 8. 用語

| 用語 | 説明 |
| --- | --- |
| フォーム | 管理画面で定義する入力フォームの単位 |
| フィールド | フォームを構成する入力項目 |
| 送信（サブミッション） | 応募者が送信した1件分のデータ |
| 確認画面 | 送信前に入力内容を表示して確認させる画面 |
| ハニーポット | ボット検出用の隠しダミー項目 |
